CHULETA DOCKERFILE - CONCEPTOS CLAVE
DEFINICIÓN Y FUNCIÓN
[cite_start]Qué es: Es un archivo de texto que contiene las "instrucciones" o la receta para construir una imagen[cite: 464].
[cite_start]Qué define: Especifica el sistema base que se va a usar, las dependencias que hay que instalar, los archivos a utilizar y cómo debe ejecutarse el programa en cuestión[cite: 256].
[cite_start]Propósito: Representa el método estándar y automatizado para crear tus propias imágenes personalizadas (lo que se conoce como dockerizar una aplicación)[cite: 360, 544, 546].

EL FLUJO DE TRABAJO (WORKFLOW)
[cite_start]La secuencia fundamental de trabajo en Docker es: Dockerfile -> build -> Docker Image -> run -> Docker Container[cite: 255, 256, 257, 258].
El proceso de construir un Dockerfile da como resultado directo una Imagen. [cite_start]Esta imagen resultante es inmutable[cite: 257, 465].

ARQUITECTURA DE CAPAS
[cite_start]En Docker existen dos formas de crear imágenes propias: a partir de un contenedor modificado (haciendo un commit) o a partir de un Dockerfile[cite: 361, 362, 363].
[cite_start]Al construir la imagen, las instrucciones del Dockerfile crean una serie de capas apiladas unas sobre otras (la superior es la más reciente), creando un sistema de archivos unificado y optimizando el espacio en disco[cite: 354, 355, 356, 465].

IMAGEN BASE (Obligatorio. Siempre es la primera línea)
Define el punto de partida (ej. ubuntu, alpine, node, python)
FROM ubuntu:22.04

VARIABLES DE ENTORNO (Opcional)
Establece variables que estarán disponibles durante la creación y ejecución
ENV ENTORNO=produccion

DIRECTORIO DE TRABAJO (Recomendado)
Crea la carpeta dentro del contenedor y se mueve a ella (como hacer un 'cd')
WORKDIR /mi_aplicacion

COPIAR ARCHIVOS (Del host al contenedor)
Sintaxis: COPY <origen_en_tu_pc> <destino_en_contenedor>
El punto (.) significa "todo lo que hay en mi carpeta actual"
COPY . /mi_aplicacion

EJECUTAR COMANDOS DE INSTALACIÓN (Se ejecuta al hacer 'docker build')
OJO: Cada instrucción RUN crea una nueva capa en la imagen [cite: 356]
RUN apt-get update && apt-get install -y python3

EXPONER PUERTOS (Informativo)
Indica en qué puerto va a escuchar la app (no lo mapea mágicamente al exterior, para eso usas -p en el docker run)
EXPOSE 8080

COMANDO POR DEFECTO (Se ejecuta al hacer 'docker run')
Es el proceso principal del contenedor (PID 1). Si este proceso muere, el contenedor se para.
CMD ["python3", "app.py"]

Ejercicio 2: Construcción de imagen propia con Dockerfile
Cambiamos a la carpeta del segundo ejercicio, creamos el archivo index.php solicitado y editamos el Dockerfile de forma automatizada:
mkdir -p ~/ejer_2_Ferrer
cd ~/ejer_2_Ferrer

echo "<?php phpinfo(); ?>" > index.php
# 2. Abrir gedit para escribir el Dockerfile
gedit Dockerfile &

Copia y pega este contenido exacto dentro del Dockerfile, guárdalo y ciérralo:
# Partiendo de la imagen base oficial solicitada
FROM php:8.2-apache

# Actualizar repositorios e instalar los paquetes requeridos de forma desatendida (-y)
RUN apt update && apt install -y default-mysql-client curl

# Instalar la extensión de PHP necesaria para interactuar con MySQL
RUN docker-php-ext-install mysqli

# Copiar el archivo index.php local al directorio correspondiente de la imagen
COPY index.php /var/www/html/

# Configurar las variables de entorno solicitadas en el examen
ENV DB_HOST=db
ENV DB_USER=root
ENV DB_PASS=examen123
ENV DB_NAME=examen_db

# Informar que la imagen escucha por el puerto 80
EXPOSE 80

# Definir el ejecutable y sus argumentos por defecto de forma flexible
ENTRYPOINT ["apache2ctl"]
CMD ["-D", "FOREGROUND"]

Una vez cerrado el editor, ejecuta la construcción de la imagen en tu terminal:  
# 3. Construir la imagen local bajo el nombre "php-web:v1"

📁 Ejercicio 3: Despliegue multicontenedor con Docker Compose
Pasamos al último ejercicio, clonamos el contenido del Ejercicio 2 dentro de este espacio (tal y como exige el enunciado) y creamos el orquestador:  

mkdir -p ~/ejer_3_Ferrer
cd ~/ejer_3_Ferrer
# 1. Copiar el entorno de construcción anterior aquí dentro
cp -r ~/ejer_2_Ferrer/. .
# 2. Abrir gedit para estructurar el escenario declarativo
gedit compose.yaml &

Copia y pega esta configuración estructurada dentro de compose.yaml, guarda y cierra:
services:

  db:
    image: mysql:8
    container_name: contenedor_db
    environment:
      - MYSQL_ROOT_PASSWORD=examen123
      - MYSQL_DATABASE=examen_db
    volumes:
      - db_data:/var/lib/mysql
    networks:
      - red3_Ferrer

  web:
    build: .  # Al haber copiado el Dockerfile aquí, se construye en el contexto local
    container_name: contenedor_web
    ports:
      - "8585:80"
    depends_on:
      - db
    networks:
      - red3_Ferrer

  adminer:
    image: adminer
    container_name: contenedor_adminer
    ports:
      - "9090:8080"
    networks:
      - red3_Ferrer

volumes:
  db_data:

networks:
  red3_Ferrer:
    driver: bridge
    ipam:
      driver: default
      config:
        - subnet: 192.168.20.0/24
          gateway: 192.168.20.254

Una vez que cierres gedit, lanza la infraestructura completa y realiza las comprobaciones finales desde tu terminal:

docker compose up -d
# 4. Comprobar que los tres contenedores se encuentran levantados (Up)
docker compose ps
